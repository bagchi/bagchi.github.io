1       Error Detection in High Throughput Distributed Applications

Project lead faculty member: Saurabh Bagchi

Problem Statement

The proliferation of high bandwidth applications and the increase in the number of consumers of distributed applications have caused them to operate at increasingly high data rates. Many of these distributed systems form parts of critical infrastructures, with real-time requirements. Downtime of a system providing critical services in power systems, air traffic control, banking, and railways signaling could be catastrophic. The errors may come from individual software components, interactions between multiple components, or misconfiguration of these components. It is therefore imperative to build low latency detection systems that can subsequently trigger the diagnosis and recovery phases leading to systems that are robust to failures. In a sub-class of detection systems called stateful error detection, the detection rules are based on the state of the application components, and thus on aggregated information rather than on instantaneous information. Though the merits of stateful detection seem to be well accepted, scaling a stateful detection system with an increasing number of application components or increasing data rate has been a challenge. This is due to the increased processing load of tracking application state and rule matching based on the state.

Project Tasks

In this project, we will address this issue through designing a runtime monitoring system focused on high throughput distributed applications. Specific tasks to be achieved will include:

1.    Sampling approaches to determine the high value messages to inspect for errors

2.    Machine learning techniques to identify abnormal behavior, without using pre-specified rules

3.    Determining patterns of application behavior that precede a failure; Such predictive property will serve as triggers for heavy-duty detection

Qualifications

Required: Development of networked applications, kernel-level networking stack, Java programming language, php scripting.

Desirable: Machine learning algorithms (clustering, pattern classification), time series analysis, web services.

2       Diagnosis in Wireless Embedded Systems with Customized Hardware Support

Project Lead Faculty Members: Vijay Raghunathan and Saurabh Bagchi

Problem Statement

Networked wireless embedded (NWE) systems, such as wireless sensor networks, are increasingly being used in civilian and military domains and in our nations critical infrastructure. However, the lack of robustness in these systems has hampered their widespread adoption for such mission-critical applications. The lack of robustness in these systems arises from naturally occurring hardware and software failures and malicious attacks. This proposal focuses on the former cause of unreliable operation. The causes of failures are manifold. They span hardware failures, software failures, or a combination of the two. The failure causes can be internal to a node (such as a race condition in the software executing on the node) or external to a node (such as a nearby electromagnetic source causing interference on the wireless channel). The environment in which NWE systems operate is inherently concurrent and asynchronous, meaning that multiple events that demand attention from a node occur close together in time, and that the events occur without predetermined schedules. Further, the software on NWE systems is distributed. These factors make it particularly challenging to ensure the reliable operation of NWE systems. This challenge manifests itself in simplistic or non-existent error-handling code interspersed with functional code in todays applications. Also current state-of-the-art cannot easily make hardware events visible to a distributed fault-tolerant software layer.

Project Tasks

In this project, we will design, develop, and evaluate a system called Charm for improving the reliability of NWE systems. Charm embodies a synergistic hardware-software co-design approach to distributed error detection and diagnosis, which will be achieved through three research thrusts: (i) Hardware support for embedded system reliability. We will design a dedicated, lightweight reliability co-processor integrated with an embedded node. Such an approach provides a logically and physically separate foundation to monitor the behavior of hardware and software at an intra-node level and easily detect anomalous events, (ii) Network-level monitoring service. This thrust deals with the design of a distributed, network-level monitoring service that uses observed event streams from individual nodes to perform detection and diagnosis of errors at runtime. Sub-thrusts within this include performing detection with incomplete observations using probablistic techniques, performing root-cause identification by determining causal dependencies through observations, and approaches to make the monitoring adaptive to the current error state (e.g., collecting more fine-grained state information on demand), and (iii) Interactions between the reliability co-processor and the network-level monitoring service. We will develop customized, low-overhead communication primitives for the network monitoring infrastructure to communicate with the reliability co-processor. This will involve creating an appropriate programming model for the reliability co-processor and investigating the appropriate choice of push, pull, or hybrid mechanism.

Our research group is responsible for the second thrust and, jointly with Prof. Raghunathans group, the third thrust.

Qualifications

Required: Programming in an embedded environment, network programming, digital hardware interfacing.

Desirable: Machine learning algorithms (clustering, pattern classification), time series analysis, sensor network programming.
